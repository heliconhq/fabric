## 9.0.0

### Major Changes

- e799fda: Accept Keycloak instance as an injected dep

  ### Why?

  It's often needed that consuming clients have access to the Keycloak instance. The most common usecase we've encountered is setting up network interceptors outside of the React tree.

  When Keycloak is encapsulated it's very hard to access it outside of React.

  ### How to update your project code

  Initialize Keycloak instance in your project and pass it to `<App/>` through the `keycloakInstance` prop.

  You may also config the initialization through the `keycloakInitOptions` prop.

  Since Keycloak is now instantiated in the consuming application, you would need to handle token refreshes in a way that works with how you manage network requests like in an axios interceptor, for example.

  Everything else should work the same.
